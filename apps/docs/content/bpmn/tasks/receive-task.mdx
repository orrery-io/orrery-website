A receive task waits for a named message — identical in behavior to a [message intermediate catch event](/bpmn/events/messages#message-intermediate-catch-event), but rendered as a task shape in the diagram.

```xml
<receiveTask id="waitForApproval" name="Wait for Approval"
  messageRef="ApprovalMessage" />
```

See [message correlation](/bpmn/events/messages#message-correlation) for the full matching algorithm.
