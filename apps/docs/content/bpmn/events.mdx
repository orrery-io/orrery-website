# Events

Events represent things that happen during a process.

## Start Events

| Type | Description |
|---|---|
| None | Standard process start |
| Timer | Process starts on a schedule |
| Message | Process starts when a message is received |
| Signal | Process starts when a signal is broadcast |

## End Events

| Type | Description |
|---|---|
| None | Normal completion |
| Error | Throws an error (caught by boundary events) |
| Terminate | Immediately cancels the entire process |
| Escalation | Throws an escalation |
| Message | Sends a message on completion |
| Signal | Broadcasts a signal on completion |

## Intermediate Events

Pause execution until a condition is met.

| Type | Description |
|---|---|
| Timer Catch | Wait for a duration or until a specific time |
| Message Catch | Wait for a specific message |
| Signal Catch | Wait for a signal broadcast |
| Intermediate Throw | Pass-through milestone (no waiting) |

## Boundary Events

Attached to tasks — trigger when an event occurs while the task is active.

| Type | Behavior |
|---|---|
| Timer | Triggers after a duration |
| Message | Triggers when a message arrives |
| Error | Triggers when a task throws an error |
| Escalation | Triggers on escalation |
