# Subprocesses

Subprocesses group activities into a reusable, nested scope.

## Subprocess

A subprocess embeds a complete process within a parent process. Variables from the parent scope are accessible.

```xml
<subProcess id="orderFulfillment" name="Order Fulfillment">
  <!-- nested BPMN elements -->
</subProcess>
```

## Event Subprocess

An event subprocess is triggered by an event while the parent process is running. It can interrupt the parent or run alongside it.

```xml
<subProcess triggeredByEvent="true">
  <startEvent>
    <errorEventDefinition errorRef="paymentError" />
  </startEvent>
</subProcess>
```

**`isInterrupting`:** If true, cancels the parent scope on trigger. If false, runs concurrently.
